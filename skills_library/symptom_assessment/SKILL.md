# symptom_assessment

## 适用场景

- 用户描述身体不适，希望了解可能原因和严重程度
- 用户不确定是否需要去医院、该挂什么科
- 用户需要用药建议或家庭护理指导

## 依赖的其他SKILL

- `user_profile`: 获取用户年龄、性别、既往病史、过敏史等
- `daily_tracking`: 获取近期睡眠、饮食、运动、情绪等日常数据
- `physiological_baseline`: 获取生理指标基线（用于对比）
- `physiological_analysis`: 获取实时生理数据（心率、血氧、体温等）

## 问询流程

应当按照以下步骤逐步引导用户体温，每次只询问少量问题，避免用户信息过载

1. 询问核心不适症状，如哪里不适，什么痛觉等
2. 细节盘问，如疼痛程度，是间隔性还是持续性，最近是否加重
3. 询问伴随症状
4. 基础情况盘问，有误特殊事件的发生
5. 结合用户基本信息、日常数据等，自动关联到其他健康SKILL获取用户健康数据进行分析

## 反馈

1. 用户病况的可能方向
2. 用户病况的紧急性，是否立马就医
3. 提出购药建议
4. 主动将本次病况问询的结果和细节写入日常记录中

## 注意事项

- 本SKILL不提供工具说明，当需要调用相关信息工具，按照相关联的SKILL说明进行
