# medical_report_analysis

## 适用场景

- 用户上传体检报告、化验单、病例报告等医疗文件，希望获得通俗解读和健康建议
- 用户希望做一次某方面的报告分析，如睡眠、运动、饮食等
- 报告中的异常指标含义、风险程度及后续行动

## 依赖的其他SKILL

- `user_profile`: 获取用户基本信息（年龄、性别、既往病史、过敏史等）
- `daily_tracking`: 获取近期饮食、运动、睡眠、用药等日常记录
- `physiological_baseline`: 获取用户生理指标基线，用于对比
- `sleep_analysis`: 获取用户睡眠数据

## 提供的工具

- `generate_medical_report`: 通过提取医疗报告信息后经过整体分析（关联用户基本数据、日常数据等），进行风险分层，提供改善方案和就医建议，或者是用户需要做一次某方面健康分析报告

  参数:

  - report_type: 报告类型(string)，针对用户的输入需求，可以是医疗文件的报告分析，也可以是基于用户需求的某方面健康分析，宽泛性类型
  - report_title: 报告摘要(string)
  - report_content: 输入参数以JSON形式的字符串类型，其中的字段消息不规定字段，按照用户需求和输入（如果有图片医疗文件）的内容拆解成合适的字段，加上通用解读的说明、建议改善方案和就医建议等

  返回值: 将健康报告保存成功与否的反馈

## 使用流程

1. 用户可以提供文本或者医疗文件做分析解读，分析解读时按需主动结合其他SKILL综合分析
2. 若报告中包含指标与用户历史数据相关，自动从其他SKILL获取用户档案和日常记录
3. 经分析后的内容不仅需要显示给用户，还需要调用 `generate_medical_report` 工具保存内容
