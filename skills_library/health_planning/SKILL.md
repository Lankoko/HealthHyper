# health_planning

## 适用场景

- 用户希望根据自身健康目标（如减重、增肌、改善睡眠、控制血压等）获得个性化行动计划
- 用户希望保存和查看自己的健康计划
- 可以按照用户需求保存非健康主题的事件计划

## 依赖的其他SKILL

- `user_profile`: 获取用户基本信息（年龄、性别、体重、身高、病史等）
- `physiological_baseline`: 获取生理指标基线（用于设定合理目标）
- `daily_tracking`: 获取用户近期日常记录（饮食、运动、睡眠），用于计划参考
- `sleep_analysis`: 获取睡眠改善建议（如果目标是改善睡眠）
- `medication_assistant`: 获取用药计划（如果需要整合用药提醒）

## 提供的工具

- `create_health_plan`: 根据用户需求制定个性化健康计划保存下来

  参数:

  - title: 计划标题(string)
  - content: 计划内容(string)，输入为字符串数据类型，但要求形式为JSON形式

- `get_health_plans`: 获取最近的计划内容

  参数:

  - days: 最近天数，天数小于3大于0

  返回值: 最近天数的计划内容

## 计划结构

健康计划包含以下模块（可根据目标调整），在创建计划时，content可以参考一下模块设计JSON形式:

- 总体目标与周期
- 饮食建议（每日热量、营养素比例、具体食物推荐）
- 运动计划（类型、频率、时长、强度）
- 作息与睡眠（固定上床/起床时间、睡前放松流程）
- 生理指标监测（如每周测体重、血压）
- 行为习惯（如每日喝水提醒、步数目标）
- 用药提醒（如果与用药计划关联）
- 每周复查与调整机制

## 使用流程

1. 用户出现制定计划需求，结合用户身体状况或具体消息生成计划后，调用 `create_health_plan` 写入
2. 用户需要查询或者其他SKILL需要查看最近计划分析时，调用 `get_health_plans` 获取最近计划
