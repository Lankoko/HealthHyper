# 一、 课题的目的和任务

1、 课题的目的

随着人们健康意识的提升和物联网、人工智能技术的普及，传统健康管理方式存在数据分散、管理不便、个性化服务不足等问题，难以满足用户实时化、智能化的个人健康管理需求。本项目旨在设计并实现一套融合穿戴式硬件、鸿蒙交互终端、云端服务、AI智能分析的个人健康管家系统，为用户提供全周期的健康管理服务。

本人负责系统中的核心模块——基于Spring Boot的健康数据业务中台的开发，通过搭建云端服务，实现全系统健康数据的统一管理、核心业务逻辑落地、多模块数据交互协同，为前端鸿蒙终端和后台AI健康引擎提供稳定可靠的数据与接口支撑，保障系统端-边-云架构顺畅运行。

# 二、硬件端：规则树与状态机，以及个人基线建立

## 1. 规则树 vs 状态机

|概念|定义|特点|适用场景|
|---|---|---|---|
|规则树|一组静态的“if-then-else”判断逻辑，呈树状分支结构|简单直观，易于实现，但无法记忆历史状态|单次异常判断 (如血氧<90%立即报警)|
|状态机|有限状态及状态间转移规则的模型，具有“记忆”能力|可处理时序过程，避免频繁触发，能区分持续异常与瞬时干扰|需要连续观察才触发的报警 (如心率异常持续5分钟)|
建议：异常提醒采用状态机，因为“不在个人基线”需要观察一段时间的偏离情况，避免因单次测量误差误报。

## 2. 个人水平基线的建立

基线是动态的，需要分阶段建立：

|阶段|操作|说明|
|---|---|---|
|初始化 (前3-5天)|记录每日静息时段的心率、血氧、SDANN，取中位数作为初始基线|此时不触发“偏离基线”报警|
|正常运行|每24小时更新一次基线 (如滑动窗口取最近7天的中位数)|适应身体状态的缓慢变化|
|异常检测|当实时值偏离基线超过设定阈值 (如心率均值±15%)，且持续超过设定时间 (如5分钟)，触发报警|用状态机实现“持续偏离”判断|
基线存储：在MCU的Flash中存储每个用户的基线参数，支持多用户切换。

## 3. 硬件端需要实现的功能清单

根据描述，硬件端需要完成以下信号处理与特征计算：

|传感器|原始信号|处理后特征|用途|
|---|---|---|---|
|MAX30102|PPG (Red/IR)|心率、血氧、HR_CV、SDANN|异常报警、上传手机|
|MPU6050|加速度 (x,y,z)|活动量、翻身次数|异常报警、上传手机|
其中 SDANN 需要5分钟以上的数据才能计算，在硬件端可每5分钟计算一次并更新。

# 三、手机端：关于输入数据形式的建议

提出的问题：输入原始IMU信号 (x/y/z原始值) 是否可行或推荐？

## 1. 两种方案对比

|方案|输入形式|优点|缺点|
|---|---|---|---|
|方案A (当前设计)|手工特征<br>[hr_mean, hr_cv, activity, turn_out]|模型极小 (参数量<1万)，推理快，可解释性强，与公开数据集直接适配|特征设计依赖先验知识|
|方案B (原始信号)|原始PPG+三轴加速度 (如30秒×4通道)|模型可自动学习更丰富特征，可能达到更高精度|输入维度暴增 (约3000点)，模型端推理负担重，需要大量训练数据|
## 2. 推荐方案

继续使用方案A (手工特征)，理由：

• 已基于PhysioNet数据集验证了该特征集的有效性，且成功转换为 .ms 模型。

• 硬件端已经需要计算这些特征用于异常报警，手机端直接复用，无需传输原始信号，节省带宽和功耗。

• 模型体积小，在鸿蒙端推理速度极快 (<10ms)，对用户体验友好。

如果未来希望提升精度，可在云端收集更多用户数据后，尝试用方案B训练新模型，但需评估手机端算力是否满足实时推理。

# 四、云端：基于LangChain的AI智能体设计

计划构建一个能够结合生理数据与对话交互的AI智能体，这是一个很有价值的扩展。核心挑战在于如何将结构化数据 (心率、睡眠分期等) 与非结构化文本 (对话、病历) 融合，并生成有医学参考价值的建议。

## 1. 整体架构

|Plain Text<br/>用户输入 (文本/语音) ──┐<br/>                     ├→ LangChain Agent ─→ 工具调用 ─→ 生成回复<br/>生理数据 (手机上传) ──┘           │<br/>                               └──→ 知识库/记忆|
|---|
## 2. 关键模块设计

|模块|功能|技术实现|
|---|---|---|
|对话上下文管理|记录用户历史对话、当前会话状态|LangChain 的 ConversationBufferMemory|
|生理数据检索工具|查询用户最近的心率、睡眠分期、异常记录|封装为LangChain Tool，从数据库查询|
|知识库问答|提供睡眠健康、用药常识等知识|使用RAG (检索增强生成)，向向量数据库存储医学知识|
|分析建议生成|结合数据和对话，生成个性化建议|构造包含数据摘要的Prompt，调用大模型|
## 3. 示例工作流：用户询问睡眠质量

|Plain Text<br/>用户: “我昨晚睡得怎么样？”<br/>↓<br/>Agent调用工具: 获取最近一晚的睡眠分期数据 (深睡时长、REM占比、睡眠效率等)<br/>↓<br/>构造Prompt: “用户昨晚深睡1.2小时，REM 1.5小时，睡眠效率85%。请根据这些数据给出睡眠质量分析和改善建议。”<br/>↓<br/>大模型生成回答: “您的深睡时长略低于推荐值 (1.5-2小时)，建议睡前减少蓝光暴露……”<br/>↓<br/>返回用户，并保存本次分析到记忆|
|---|
## 4. 技术实现建议

• 框架：LangChain + 任意大模型API (如OpenAI、百度文心、智谱等，注意医疗合规性)

• 数据存储：用户生理数据存入时序数据库 (如InfluxDB)，对话记录存入关系数据库

• 部署：可用FastAPI封装云端服务，手机端通过HTTP协议调用

• 隐私与安全：生理数据属于敏感信息，传输加密，用户授权管理
