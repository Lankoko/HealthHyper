# sleep_analysis

## 适用场景

- 用户查询单晚或连续多晚的睡眠质量分析。
- 用户希望获得睡眠改善建议、个性化睡眠计划。
- 其他健康分析SKILL需要睡眠数据作为分析依据。

## 依赖的其他SKILL

- `physiological_baseline`: 获取心率、血氧、体温等生理指标的个性化基线
- `physiological_analysis`: 获取平时生理数据（心率、血氧、体温）的统计值
- `daily_tracking`: 获取可能影响到睡眠的相关日常记录
- `user_profile`: 获取用户基本信息（年龄、性别、生活习惯等）

## 提供的工具

- `obtain_single_night_sleep`: 获取最近一晚的睡眠数据供于分析

  参数: 无参数输入
  返回值: 字符类型数据，记录用户睡眠情况的数据，包含不同睡眠阶段占比，当晚睡觉时间、睡觉时长、相关生理参数统计值

- `obtain_multi_night_sleep`: 获取最近几晚的睡眠数据供于分析

  参数: 

  - days: 获取天数，天数在7天以内

  返回值: 字符类型数据，提供每晚睡眠的数据

- `get_sleep_plan`: 获取用户的睡眠改善计划

  参数: 无参数输入

  返回值: 字符类型数据，提供用户最新的睡眠安排

- `update_sleep_plan`: 根据最新睡眠分析更新睡眠计划 

  参数: 

  - items_json: 输入参数以JSON形式的字符串类型，其中JSON的字段主要为index、desc、category等，可以动态增加合适的字段记录，这里提供输入示例:
  ```
  {
    {"index":  0, "desc":  "每天运动健身30分钟", "category": "exercise"},
    {"index":  1, "desc":  "睡前1小时不接触电子产品", "category": "sleep"}
  }
  ```

  返回值：字符类型数据，更新成功与否

## 使用流程

1. 用户需要进行每晚睡眠的分析时，调用 `obtain_single_night_sleep` 获取睡眠相关的数据再做分析
2. 需要关注到用户最近连续几天的睡眠情况时，调用 `obtain_multi_night_sleep` 获取连续几晚睡眠数据再做分析
3. 提供用户改善睡眠方式，先调用 `get_sleep_plan` 查看最近用户睡眠计划是否合适，如果不合适或没有，则基于最近7天睡眠数据分析生成计划
4. 调用 `update_sleep_plan` 更新用户睡眠计划

## 分析用户睡眠数据知道

### 心率分析

- 夜间心率是否在基线范围内（侧重自主神经功能）
- 心率波动是否异常（反映睡眠连续性）
- 与历史趋势对比

### 血氧分析

- 夜间血氧是否稳定在正常区间（侧重睡眠呼吸暂停风险）

### 体温分析

- 夜间体温下降幅度和节律是否正常（影响入睡和睡眠深度）

### 睡眠阶段分析

- 总睡眠时长、各阶段占比（深睡、REM、浅睡）
- 深睡不足是否导致身体修复不足
- REM不足影响记忆情绪调节受损
- 浅睡过多是否导致睡眠碎片化

### 综合评级

- 优秀/良好/一般/差
- 主要问题定位（入睡难、易醒、睡眠不足、作息紊乱、生理异常等）
- 风险提示（神经紧张、压力大、呼吸暂停倾向等）

### 行为调整建议

- 针对心率偏高/血氧偏低/深睡不足等具体问题
- 作息与节律调整（固定生物钟）
- 就医建议（严重异常时）

### 环境与习惯优化

- 推荐室温、湿度、遮光/白噪音方案
- 晚餐内容建议
- 睡前放松流程（渐进式肌肉放松、正念冥想）

### 睡眠计划

- 根据用户基本信息和近期数据制定
- 动态调整机制
